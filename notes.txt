WA GTM Project — Deployment Readiness Plan
Background
After a full audit of every source file in the project (35 files across dashboard, worker, shared package, migrations, and config), the core architecture is sound. The codebase is clean, well-structured, and follows good patterns. However, 5 critical bugs and several production-readiness gaps were discovered that must be fixed before deployment.

User Review Required
CAUTION

Critical: Reconnect Loop Storm — The "Test Account" (+1234567890) is caught in an infinite reconnect loop (Status 440 → reconnect → 440 → reconnect) consuming worker resources. This is because it's a fake number that can't actually authenticate. It eventually gets a 401 (logged out), but not before wasting dozens of reconnect cycles. This account must be removed and the reconnect logic needs exponential backoff.

WARNING

Hydration Mismatch — The dashboard is throwing React hydration errors on AccountCard. Root cause: new Date().toLocaleString() renders differently on server vs client (different timezone/locale). This causes a visual flash and console errors in production.

IMPORTANT

Realtime Subscription Timeout — Worker's Supabase Realtime channel reports TIMED_OUT status, which means dynamic account scaling (add/remove accounts via UI → worker picks them up) is unreliable. The worker needs a heartbeat/reconnect mechanism.

Component 1: Worker — Critical Bug Fixes
[MODIFY] 
client-manager.js
Bug 1 — No Exponential Backoff on Reconnect:

Line 196: Math.pow(2, 0) is always 1, so delay is always 5s. Should track retry count per account.
Fix: Add a retryCount map and implement real exponential backoff with a max of 5 retries before marking as disconnected.
Bug 2 — No Realtime Reconnect on Timeout:

The Realtime subscription times out silently. Need to handle TIMED_OUT and CLOSED statuses and resubscribe.
Bug 3 — No Graceful Handling of Status 440 (Connection Replaced):

Status 440 means "Connection replaced" (another device logged in). Currently treated as reconnectable, which causes a loop. Should mark as disconnected instead.
Improvements:

Add max retry limit (5 attempts) before giving up
